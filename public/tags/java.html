Connection securisé et reverse proxy

Dans le cadre d’un projet, j’ai une configuration Spring Secure tel que : ... http.requiresChannel() .antMatchers("/client/").requiresSecure() .antMatchers("/fr/client/").requiresSecure() .antMatchers("/es/client/").requiresSecure() .antMatchers("/en/client/").requiresSecure() ... Ce qui provoque un redirect vers https si l’on tente d’accéder à l’une de ces URL en http. Le problème est lorsque que je mets un reverse proxy NginX devant, c’est lui qui s’occupe du https, il redirige vers mes routes Spring en http. Spring ne permet de configurer qu’un seul port de serveur qui peut faire http/https. »

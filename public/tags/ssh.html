Authentification SSH par certificat

L’objectif est de pouvoir se connecté via SSH à un serveur sans fournir de mot de passe. Cela ne sera bien-sur possible de depuis une machine ayant la clé privé d’installé ! On commence par générer la paire de clés si c’est pas déjà fait : ssh-keygen -t rsa -C "your_email@example.com" Laisser les clés se générer dans l’emplacement par défaut et laisser la passphrase vide sinon il faudra à chaque fois renseigner la passphrase. »

Download fichier via SSH avec reprise

La plus part du temps on utilise SCP mais sur des fichiers de taille conséquente, il est pratique de pouvoir reprendre le téléchargement si la connexion a été rompue ou autres. RSync permet de faire ça. rsync --partial --progress --rsh=ssh user@1.1.1.1:/home/user/database.tar.gz database. »

NFS au travers de SSH

Faire passer du NFS au travers du SSH n’est pas si simple que ça. Il faut : Pré-requis Vérifier que le server SSH autorise l’IP Forwarding Configuration du client SSH Il faut faire une redirection de port. Pour ça, soit la ligne de commande : ssh login@server-ssh -p 2222 -L 3049:server-nfs:2049 -L 3045:server-nfs:627 -N -f Le -N permet de ne pas proposer l’invite de commande, le -f lance le process SSH en background. »
